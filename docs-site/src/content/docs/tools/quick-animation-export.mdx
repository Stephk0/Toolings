## At a glance

Exports a separate FBX per action, driven by the NLA and action filters. Pick the actions you want, set a naming pattern, click export. Used for character/creature pipelines where every locomotion and reaction state is a discrete clip.

## Install

<Steps>
1. Download `QuickAnimationExport_v1.0.9.zip`.
2. **Edit → Preferences → Add-ons → Install from Disk**.
3. Enable. Look for the **QAE** tab in the N-panel while an armature is active.
</Steps>

## Day-to-day vs Advanced

<Tabs>
<TabItem label="Day-to-day">

1. Select the armature.
2. The panel lists all actions on the armature. Tick the ones you want to export.
3. Set the **output folder**.
4. Click **Export Selected Actions**.

Each ticked action is pushed to the armature, baked over its action range, and written as `<armature>_<action>.fbx`.

</TabItem>
<TabItem label="Advanced">

### Naming pattern

The default name is `{armature}_{action}.fbx`. The **Name Pattern** field accepts tokens:

- `{armature}` — armature object name
- `{action}` — action data-block name
- `{index}` — 0-padded sequence number
- `{prefix}` / `{suffix}` — values from the corresponding fields

Example pattern: `anim_{action}_{index}.fbx` → `anim_idle_00.fbx`, `anim_run_01.fbx`, …

### Action filters

Filter the action list by:

- **Prefix** — only show actions starting with a given prefix (e.g. `anim_`).
- **Selected only** — hide actions not currently pushed to the NLA.
- **Recently modified** — show actions edited in the last session.

### Frame range source

Pick where each clip's frame range comes from:

- **Action range** (default) — uses the action's keyframe start/end.
- **Scene range** — uses the scene timeline for every clip.
- **NLA strip range** — when the action is on the NLA, uses that strip's range.

### Subset selection

For armatures with hundreds of actions, the **Filter by Tag** option groups actions by custom property tags. Useful for shipping just `combat_*` while skipping `idle_*` in one pass.

</TabItem>
</Tabs>

## Troubleshooting

<Aside type="tip" title="One action keeps overwriting another">
Two actions resolving to the same output filename collide. Check the **Name Pattern** — `{armature}_{action}.fbx` is unique only if action names are unique. Add `{index}` to disambiguate.
</Aside>

<Aside type="caution" title="Bone constraint baking">
Like ALQE, this exporter relies on Blender's FBX export step to bake bone constraints. Drivers and complex constraint setups can produce unexpected output. Bake constraints to keyframes first if the engine doesn't match the viewport.
</Aside>
