## At a glance

You have an armature with multiple NLA tracks layered together — base locomotion, an additive aim pose, an additive lean. You want a single FBX that bakes the visible result into one clean action, without permanently destroying the layer setup. This addon does exactly that, in one click.

It's beta because edge cases around bone constraints and IK chains are still being shaken out. The common case (additive + override layers on a humanoid rig) works.

## Install

<Steps>
1. Download `AnimLayersQuickExport_v0.3.0.zip`.
2. **Edit → Preferences → Add-ons → Install from Disk**.
3. Enable. The panel lives in the N-panel under **ALQE** while an armature is active.
</Steps>

## Day-to-day vs Advanced

<Tabs>
<TabItem label="Day-to-day">

1. Select the armature.
2. Set the **export path**.
3. Click **Quick Export**.

The result is `<armature_name>.fbx` containing the flattened animation across the scene's frame range. Source NLA setup is untouched.

</TabItem>
<TabItem label="Advanced">

### Frame range overrides

The default uses the scene's frame range. Override with explicit Start/End fields when you want to export only a sub-range without resizing the timeline.

### Sample every Nth frame

For ultra-tight runtime budgets, set **Sample Step** > 1. The bake samples every Nth frame and interpolates between them. Default is `1` (sample every frame).

### Bake-only vs bake + clear

By default, the bake produces a new action and the NLA stack is preserved. The **Replace Active Action** option promotes the baked action to the armature's current action and clears the NLA — useful when the layered state was scratch work.

### Object-space root motion

The **Bake Root Motion** option flattens the armature object's world transform into the root bone's animation. Required for some game-engine ingestion pipelines that ignore the object transform.

</TabItem>
</Tabs>

## Troubleshooting

<Aside type="caution" title="IK targets not exported">
FBX export drops dependency relationships. If your rig uses Empty objects as IK targets, the FBX side won't have them. Either bake the IK chains to FK first, or include the targets in the export selection.
</Aside>

<Aside type="tip" title="Result looks wrong in engine">
Engines disagree about FBX axis conventions. The export uses Blender's default `-Z forward, Y up` for the armature; switch via Blender's FBX export panel if your engine needs something else.
</Aside>

<Aside type="caution" title="Beta status">
This addon is beta because constraint-heavy rigs (spline IK, dynamic constraints) haven't been thoroughly tested. Back up before running on production rigs.
</Aside>
